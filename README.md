# WRSGwiggle v1.0

WRSGwiggle is an ultra-lightweight, HTML-style semantic injection language. It acts as an asynchronous data binding layer that enables you to pull raw custom text extensions directly into your HTML document structure and pair them with dynamic rendering scripts—completely bypassing browser security restrictions and complex workflow pipeline configurations.

## How to Learn the Language

WRSGwiggle relies on a clean, HTML-like layout tag syntax using custom semantic elements. The entire language framework consists of exactly two tags:

1. **`<wrsg-wiggle src="YOUR_FILE"></wrsg-wiggle>`**: Specifies the target source text asset (e.g., `.doctor`, `.txt`, `.md`, or any proprietary file extension) that you want to inject natively into the page.
2. **`<doggy-engine src="YOUR_SCRIPT"></doggy-engine>`**: Binds the custom processing library script that will format, style, and structure the raw string content once it lands inside the browser document body.

---

## Universal Usage Guide (Beyond Doctor)

You can use WRSGwiggle to stream **any** text layout format into your personal web space. For example, if you want to stream raw markdown log notes or a clean plain-text handbook file into an active layout engine, write this 3-line pattern inside an `index.html` file:

```html
<!-- Connect any raw data source -->
<wrsg-wiggle src="changelog.txt"></wrsg-wiggle>

<!-- Connect any rendering script or layout processor -->
<doggy-engine src="https://jsdelivr.net"></doggy-engine>

<!-- Load the active WRSGwiggle runtime connector engine -->
<script src="https://jsdelivr.net"></script>
```

### Direct Server Integration
Ensure your destination project repository has its deployment **Source** dropdown set directly to **Deploy from a branch** in your GitHub Pages dashboard settings. WRSGwiggle resolves the file headers asynchronously straight inside the browser memory, removing the need for any hidden automated `.yml` build configurations.
